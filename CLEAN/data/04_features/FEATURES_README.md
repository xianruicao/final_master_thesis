# `film_features_*` — what each file is and how it's made

There are **5** feature tables in this folder. They fall into two layers:

- **Master tables** — one row *per protagonist per film*, generated by notebooks.
  Co-lead films (Monsters Inc.) have 2 rows; `soul_2020` is still present.
- **`_n17` working tables** — the analysis-ready 17-film set, derived from the
  master tables by applying the study "conventions" (drop Soul, one lead per film).

## The files

| file | rows | one row = | generated by |
|---|---|---|---|
| `film_features_all.csv` | 19 | protagonist × film: network shape, gender homophily, centrality, removal analysis | `notebooks/06_network_analysis_PAU.ipynb` |
| `film_features_extended.csv` | 19 | same key, extra ego-network / structural metrics | `notebooks/07_compute_extended_features.ipynb` |
| `film_features_social.csv` | 18 | same key, "social" metrics (monologue %, isolation %, best friend, who-addresses-whom) | `notebooks/08_compute_social_features.ipynb` (or `scripts/build_film_features_social.py`) |
| `film_features_all_n17.csv` | 17 | **one lead per film** — analysis-ready filter of `film_features_all.csv` | `analysis/h1_homophily/n17_orchestrator.py` |
| `film_features_extended_n17.csv` | 17 | analysis-ready filter of `film_features_extended.csv` | `analysis/h1_homophily/n17_orchestrator.py` |

## Pipeline (how a row gets built)

```
screenplay
  → 05_addressee_tagging_scene_level.ipynb   (tag who each line is spoken to)
  → 06_network_analysis_PAU.ipynb            → film_features_all.csv      (master)
  → 07_compute_extended_features.ipynb       → film_features_extended.csv
  → 08_compute_social_features.ipynb         → film_features_social.csv
  → analysis/h1_homophily/n17_orchestrator.py
        apply_conventions()                  → *_n17.csv  (17-film working set)
```

Each notebook **upserts** one film's row at a time (keyed by `film_id, protagonist`),
so a master table can carry leftovers from earlier runs — that's why the `_n17`
filter exists.

## Why master = 19 rows but `_n17` = 17

`film_features_all.csv` holds **18 films** (the 17 study films + `soul_2020`) and
Monsters Inc. appears twice (Mike + Sulley) → 19 rows.

`apply_conventions()` in `n17_orchestrator.py` reduces this to exactly **17 rows,
one per film**:

1. **Drop `soul_2020`** — not part of the final 17-film set.
2. **Monsters Inc. → keep Sulley only** (drop the Mike row).
3. **Drop stale co-lead / alias rows** an upsert may have left behind
   (Inside Out `Riley`, Zootopia `Hopps.`, Incredibles 2 `Elastigirl`).
4. Add two convenience columns: `film_title`, `row_label`.

So the `_n17` files = the source master table, soul-free, one lead per film,
plus the 2 display columns. **Analysis code should read the `_n17` files.**
